    What distinguishes normal HTML elements from semantic HTML elements?

    Semantic HTML elements provide more meaning to both the browser and the developer by conveying the purpose or structure of the content they wrap. For example, instead of just using a <div> to create a generic container, you might use a <header>, <nav>, <main>, or <footer> element to provide clearer structure and semantics to your document. This not only improves accessibility but also helps search engines better understand the content.

    Explain the disparity between closing tags and self-closing tags in HTML.

    Closing tags are used to mark the end of an HTML element, while self-closing tags are used to represent elements that don't have any content within them and therefore don't require a separate closing tag. For example, <img> is a self-closing tag because it doesn't contain any content; it simply inserts an image into the document. On the other hand, <p> is a tag that requires both an opening and a closing tag because it contains text or other elements between them.

    What distinguishes inline elements from block elements in HTML?

    Inline elements are those that don't start on a new line and only take up as much width as necessary. Examples include <span>, <a>, and <strong>. Block elements, however, start on a new line and take up the full width available, pushing subsequent content onto a new line. Examples of block elements include <div>, <p>, and <h1> to <h6> </p>
